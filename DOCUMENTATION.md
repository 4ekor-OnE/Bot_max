# Полная документация проекта MAX Support Bot

Версия документа: март 2026. Репозиторий: [github.com/4ekor-OnE/Bot_max](https://github.com/4ekor-OnE/Bot_max).

## 1. Назначение

Система службы поддержки для мессенджера **MAX**: пользователи создают **заявки** (тикеты), специалисты обрабатывают их, директор получает отчёты. Администратор настраивает справочники и документы через **`/admin`** в боте или через **веб-панель «Панель управления»** (Flask).

Стек: **Python 3.12+**, **SQLAlchemy 2**, **SQLite** (или другой движок через `DATABASE_URL`), **maxapi** (long polling), **Flask** (веб-интерфейс).

## 2. Структура каталогов

| Путь | Назначение |
|------|------------|
| `bot.py` | Точка входа бота, регистрация хендлеров maxapi, маршрутизация callback и текста |
| `config.py` | Загрузка `.env`, пути к данным, лимиты, настройки веб-панели |
| `installer.py` | Интерактивная первичная настройка (`.env`, проверка токена) |
| `cleanup.py` | CLI: очистка старых данных, суточная статистика |
| `models/` | ORM-модели и `database.py` (`init_db`, миграции SQLite/PostgreSQL на уровне `ALTER`) |
| `app/` | Логика админки в боте: `admin_panel.py`, `admin_documents_flow.py`, `admin_tickets_admin.py`, `admin_system_flow.py`, `fsm.py` |
| `services/` | Уведомления, файлы инструкций, фото заявок, статистика, очистка |
| `utils/` | Безопасный ответ в чат, лимиты текста, настройки из БД, callback ack |
| `keyboards/` | Сборка inline-клавиатур |
| `web_admin/` | Flask-приложение: `app.py`, шаблоны Jinja2 в `templates/` |
| `data/instructions/` | Файлы документов инструкций (не коммитятся, кроме `.gitkeep`) |
| `data/temp/` | Временные файлы |
| `data/ticket_photos/` | Копии фото заявок для отображения в панели управления |

## 3. Переменные окружения

Файл-образец: **`.env.example`**. Основные переменные:

| Переменная | Описание |
|------------|----------|
| `BOT_TOKEN` или `MAX_BOT_TOKEN` | Токен бота MAX (обязательно) |
| `ADMIN_PASSWORD` | Пароль `/admin` и веб-панели (dev); в production предпочтительно `ADMIN_PASSWORD_HASH` |
| `ADMIN_PASSWORD_HASH` | SHA-256 пароля в hex (нижний регистр) |
| `DATABASE_URL` | По умолчанию `sqlite:///support_bot.db` в каталоге проекта |
| `MAX_MESSAGE_TEXT_LENGTH` | Лимит символов для полей ввода (по умолчанию 4000) |
| `INSTRUCTIONS_DIR`, `TEMP_FILES_DIR` | Каталоги данных |
| `INSTRUCTION_MAX_BYTES` | Макс. размер файла инструкции при загрузке |
| `TICKET_PHOTOS_DIR`, `TICKET_PHOTO_MAX_BYTES` | Каталог и лимит для сохранённых фото заявок |
| `WEB_ADMIN_SECRET_KEY` | Секрет сессий Flask (обязателен в production) |
| `WEB_ADMIN_HOST`, `WEB_ADMIN_PORT` | Хост/порт веб-панели (по умолчанию `127.0.0.1:5000`) |
| `WEB_ADMIN_DEBUG` | `1` / `true` — режим отладки Flask (не на production) |

Секреты и `.env` в репозиторий не попадают (см. `.gitignore`).

## 4. Модель данных (кратко)

- **users** — пользователи: `max_id`, роль (`user` / `support` / `director`), флаги уведомлений (`notify_new_tickets`, `notify_urgent_only`, …).
- **shops**, **categories** — справочники для заявок; у категории есть **SLA** в часах.
- **tickets** — заявка: магазин, категория, автор, заголовок, описание, статус, приоритет, срочность, исполнитель, **`photo_path`** (URL или префикс `local:` + имя файла на диске), SLA-дедлайн.
- **ticket_comments** — комментарии к заявке (в т.ч. системные).
- **sessions** — FSM в JSON (состояние создания заявки, шаги админки и т.д.).
- **instruction_documents** — метаданные загруженных инструкций.
- **system_settings** — единая строка настроек (SLA, сроки хранения при очистке).
- **daily_statistics** / связанные — агрегаты для отчётов (см. `cleanup.py`, `services/statistics_rollup.py`).

Инициализация: **`init_db()`** из `models/database.py` (вызывается при старте `bot.py` и при запуске `web_admin`).

## 5. Роли и сценарии в боте

### 5.1. Пользователь (`user`)

- Главное меню: создание заявки, срочная заявка, мои заявки, инструкции, уведомления.
- **Создание заявки**: FSM — магазин → категория → заголовок → описание → опционально фото → **экран подтверждения** (`ticket_confirm_submit` / `ticket_confirm_cancel`).
- **Фото**: обработчик извлекает URL/token из вложения maxapi, при возможности **скачивает файл** и сохраняет в `data/ticket_photos/`, в БД пишется `local:<имя_файла>` или прямой `https://...`.

### 5.2. Специалист (`support`)

- Списки заявок по статусам, карточка заявки, взять в работу, решить, отложить, комментарий, назначение (из админ-режима заявок).

### 5.3. Директор (`director`)

- Отчёт за период: выбор формата CSV / Excel / оба, ввод дат `ГГГГ-ММ-ДД`.

### 5.4. Администратор по паролю (`/admin <пароль>`)

Не требует роли в БД до входа; после успешного пароля — режим **`admin_mode`** в FSM. Доступны разделы: пользователи, магазины, категории, все заявки с фильтрами, документы инструкций, настройки, очистка.

## 6. Важно: callback payload

- Кнопки открытия заявки пользователем: **`ticket_<число>`** (только цифры после префикса).
- Подтверждение создания: **`ticket_confirm_submit`** / **`ticket_confirm_cancel`** — не должны перехватываться веткой «открыть заявку»; в коде используется условие `startswith("ticket_") and callback_data[7:].isdigit()`.

Документы для пользователей: **`instr_doc_<id>`**; текстовые пункты справки — отдельные payload (`instruction_create`, …), без общего префикса с `int(...)`.

## 7. Уведомления

Модуль **`services/notification_service.py`**: push специалистам о новой заявке, о комментарии, о назначении, пользователю о смене статуса. Перед `send_message` проверяется корректность **`max_id`** (пустой или нечисловой — запись пропускается, в лог предупреждение).

## 8. Ответы в чат (`utils/safe_reply.py`)

Сначала вызывается **`message.answer(...)`**; при ошибке — **fallback** на **`bot.send_message(user_id=...)`** (актуально после нажатий inline-кнопок в MAX).

Подтверждение callback без дублирования клавиатуры: **`utils/callback_ack.py`** (невидимый символ в `notification`).

## 9. Веб-панель «Панель управления»

Запуск: **`python -m web_admin`** из корня проекта.

- Авторизация: тот же пароль, что у **`/admin`**.
- Сессия + CSRF-токен в формах.
- Разделы: сводка, пользователи (роли), магазины, категории, заявки (список, карточка, смена статуса, комментарии), просмотр фото: прямой URL или маршрут **`/ticket-photos/<filename>`** для `local:*`.

Документы инструкций в веб-панели **не дублируются** — только загрузка/список в боте (`/admin` → документы).

## 10. Запуск и эксплуатация

### Бот

```bash
cd max_support_bot
.venv/bin/pip install -r requirements.txt
cp .env.example .env   # заполнить
.venv/bin/python bot.py
```

Логи при типовом запуске с перенаправлением: **`tail -f /tmp/bot.log`**.

### Веб-панель

```bash
WEB_ADMIN_HOST=0.0.0.0 .venv/bin/python -m web_admin
```

Для доступа извне обязательны firewall, по возможности **HTTPS** за reverse proxy (см. **`DEPLOY.md`**).

### Очистка и статистика

```bash
.venv/bin/python cleanup.py --all
.venv/bin/python cleanup.py --stats-yesterday
```

## 11. Безопасность

- Не публиковать `.env`, токены, хеши паролей.
- В production: `ADMIN_PASSWORD_HASH`, сильный `WEB_ADMIN_SECRET_KEY`, ограничение доступа к порту панели.
- Файлы в `data/ticket_photos/` отдаются только авторизованным пользователям панели; имена файлов валидируются regex.
