# Инструкция по тестированию функций специалиста ТП

## Шаг 1: Узнайте свой max_id

Ваш `max_id` - это ваш уникальный идентификатор в MAX мессенджере.

### Способ 1: Через бота
1. Откройте бота в MAX мессенджере
2. Отправьте команду `/start`
3. Проверьте логи бота:
   ```bash
   tail -f /tmp/bot.log | grep max_id
   ```
   Вы увидите строку типа: `max_id из from_user: 21447227`

### Способ 2: Через скрипт
Запустите скрипт настройки - он покажет инструкцию.

## Шаг 2: Настройте роль SUPPORT

Запустите скрипт настройки:

```bash
cd max_support_bot   # каталог клона репозитория
source .venv/bin/activate   # или: python3 -m venv .venv && source .venv/bin/activate
python3 test_support_setup.py
```

Скрипт:
- Спросит ваш max_id (или использует тестовый)
- Создаст/обновит пользователя с ролью `SUPPORT`
- Создаст тестовые заявки (если их нет)

## Шаг 3: Проверьте работу функций

### 3.1. Откройте бота
1. Откройте бота в MAX мессенджере
2. Отправьте `/start`
3. Вы должны увидеть меню специалиста ТП:
   - 🆕 Новые заявки
   - ⚙️ В работе (в списке — заявки **в работе и отложенные**, назначенные на вас)
   - ⏸️ Отложенные
   - 🏪 По магазинам
   - 📊 Моя статистика
   - 🔔 Уведомления

### 3.2. Проверьте "Новые заявки"
1. Нажмите "🆕 Новые заявки"
2. Должен появиться список новых заявок
3. Нажмите на любую заявку
4. Должны увидеть:
   - Детали заявки
   - Кнопку "✅ Взять в работу"
5. Нажмите "✅ Взять в работу"
6. Заявка должна быть назначена на вас и перейти в статус "В работе"

### 3.3. Проверьте "В работе"
1. Нажмите "⚙️ В работе"
2. Должны увидеть заявки **в работе и отложенные**, назначенные на вас (отложенные помечены ⏸️)
3. Нажмите на заявку
4. Для статуса «в работе» должны быть кнопки:
   - "✅ Решить" — статус «Решена»
   - "⏸️ Отложить" — статус «Отложена»
5. После «Отложить» снова откройте "⚙️ В работе" — та же заявка **должна остаться в списке** (не пропадать)
6. В карточке отложенной заявки: "✅ Решить", "⚙️ Вернуть в работу"; «Назад» ведёт в список «В работе»
7. Если у заявки есть фото: после текста карточки приходит **отдельное сообщение с изображением**

### 3.3.1. Проверьте «⏸️ Отложенные»
1. Нажмите "⏸️ Отложенные"
2. Должны отображаться только отложенные заявки (ваши как исполнителя)

### 3.4. Проверьте "По магазинам"
1. Нажмите "🏪 По магазинам"
2. Должен появиться список магазинов с количеством **активных** заявок (новые + в работе + **отложенные**)
3. Нажмите на магазин
4. Должны увидеть список заявок этого магазина (в т.ч. отложенные с меткой ⏸️)
5. Можете выбрать заявку и взять её в работу

### 3.5. Проверьте "Моя статистика"
1. Нажмите "📊 Моя статистика"
2. Должны увидеть:
   - Всего заявок
   - Решено
   - В работе
   - Отложено
   - Процент решенных
   - Среднее время решения (если есть решенные)

## Шаг 4: Создайте тестовые заявки (опционально)

Если нужно больше тестовых данных:

1. Создайте обычного пользователя (отправьте `/start` с другого аккаунта)
2. Создайте несколько заявок через меню "📝 Создать заявку"; для проверки фото — прикрепите изображение на шаге фото
3. Эти заявки появятся в разделе "🆕 Новые заявки" для специалиста; в push о новой заявке и в карточке должно отображаться **фото** (если оно было сохранено)
4. Под пользователем откройте «📋 Мои заявки» → заявку со статусом «Отложена» после отложения специалистом — статус и фото должны быть видны

## Проверка логов

Если что-то не работает, проверьте логи:

```bash
tail -f /tmp/bot.log
```

Ищите ошибки и предупреждения.

## Быстрая проверка через SQLite (опционально)

Можно проверить данные напрямую в базе:

```bash
cd max_support_bot
sqlite3 support_bot.db

# Проверить пользователей
SELECT id, max_id, role, first_name FROM users;

# Проверить заявки
SELECT id, title, status, assigned_to FROM tickets;

# Выйти
.quit
```

## Возможные проблемы

### Проблема: Меню не изменилось на меню специалиста
**Решение:** 
- Убедитесь, что скрипт `test_support_setup.py` выполнен успешно
- Проверьте, что ваш max_id правильный
- Проверьте в базе: `SELECT max_id, role FROM users WHERE max_id = 'ВАШ_ID';`

### Проблема: Нет новых заявок
**Решение:**
- Запустите `init_data.py` для создания магазинов и категорий
- Создайте заявки через обычного пользователя
- Или скрипт `test_support_setup.py` создаст тестовые заявки

### Проблема: Ошибка при назначении заявки
**Решение:**
- Проверьте логи: `tail -f /tmp/bot.log`
- Убедитесь, что бот запущен: `ps aux | grep bot.py`
