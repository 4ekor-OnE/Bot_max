# Передача проекта заказчику — MAX Support Bot

## Что это

Чат-бот службы поддержки для мессенджера MAX: заявки, роли (пользователь / специалист / директор), админ-панель по паролю, уведомления, отчёты. Стек: Python 3, SQLite, SQLAlchemy, библиотека maxapi.

## Состав поставки (исходники)

Передайте **каталог проекта целиком**, кроме перечисленного в разделе «Не передавать». Обязательно:

| Путь | Назначение |
|------|------------|
| `bot.py` | Точка входа, long polling |
| `config.py`, `.env.example` | Настройки и шаблон переменных окружения |
| `requirements.txt` | Зависимости |
| `app/`, `models/`, `services/`, `keyboards/`, `utils/`, `web_admin/` | Логика, БД, клавиатуры, утилиты, веб-админка Flask |
| `cleanup.py` | Очистка по cron |
| `installer.py` | Первичная настройка `.env` и проверка токена |
| `README.md`, `DEPLOY.md`, `TZ_COMPLIANCE_REPORT.md`, `ПЛАН_ПРОВЕРКИ_ТЗ.md`, этот файл | Документация |

## Не передавать (или вычистить перед архивом)

- `.venv/` — виртуальное окружение (заказчик создаёт своё).
- `.env` — **секреты** (токен бота, пароль админа). Вместо этого передать только **`.env.example`** с пустыми/примерными значениями.
- `__pycache__/`, `*.pyc`
- По желанию: файл БД `*.db`, если на нём тестовые персональные данные — лучше пустая БД после `init_db` на стороне заказчика.

## Запуск у заказчика

```bash
cd max_support_bot
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
cp .env.example .env
# Заполнить BOT_TOKEN, ADMIN_PASSWORD или ADMIN_PASSWORD_HASH
.venv/bin/python bot.py
```

Подробности развёртывания и переменные — в `README.md` и `DEPLOY.md`.

## Поведение, важное для эксплуатации

1. **Токен** — в `.env`; без него бот не стартует.
2. **Админ** — `/admin <пароль>`; пароль задаётся в окружении, не хранится в коде.
3. Если пользователь **удалил сообщение** с inline-клавиатурой и снова нажал на старую кнопку, MAX может прислать callback **без привязанного сообщения**. Бот отвечает **новым сообщением в личку** по `user_id` (модуль `utils/safe_reply.py`), чтобы не «молчать».
4. Режим работы бота с MAX — **long polling**. **Webhook** для приёма событий от MAX в репозитории не поднимается (см. `TZ_COMPLIANCE_REPORT.md`). Отдельно есть **веб-админка в браузере** (`python -m web_admin`) — это не webhook, а панель управления той же БД.
